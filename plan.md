# 직무연수 이수자 영수증 발급 관리 시스템 — 워크플로우 계획

## 배경 및 요구사항

- 연구회 회비 환급을 위해 매월 직무연수 실시 (5월~11월, 월 2개)
- 1인당 최대 2개 연수 영수증 발급 가능 (각 5만원)
- 2회 초과 수강자는 이수증만 발급, 영수증 발급 없음
- 전체 명단 약 1,600명 관리 (구글 스프레드시트 `명단` 탭)
- 나이스 자동 등재를 위해 외부 기관에 이수자 명단(xlsx) 발송 필요
- 연수이수증·영수증은 구글 드라이브 클라우드에서 수강자가 다운로드

---

## 최종 확정 워크플로우

- 구현 형태는 **Tauri 기반 로컬 데스크톱 앱**으로 통일한다.
- 담당자는 앱에서 엑셀 명단을 선택하고, 대조 결과를 확인한 뒤, 영수증 발급·업로드·시트 갱신을 단계별로 실행한다.
- 자동화는 “무검증 일괄 실행”이 아니라, 각 단계의 결과를 담당자가 확인할 수 있는 방식으로 제공한다.

### 1단계 — 이수자 명단 확정 (수동)
- 담당자가 연수 이수자 명단을 확정하여 **엑셀 파일(.xlsx)** 형식으로 공유
- 입력 엑셀은 A열부터 다음 순서로 구성됨: `연번 | 연수과정명 | 성명 | 학교명 | 연락처 | 접속시간 | 이수결과 | 비고`
- 앱은 `이수결과` 값이 **"이수"**인 행만 필터링하여 이후 업무 처리 대상으로 사용
- 자동화 불가 영역 — 사람이 직접 검수 후 전달
- ⚠️ 성명·학교명·연락처 정확성 확인 철저 (이후 단계 매칭 기준이 됨)

---

### 2단계 — `명단` 탭 대조 및 발급 대상 필터링 (앱 자동화)
- 공유받은 이수자 명단과 구글 스프레드시트 `명단` 탭을 **이름 + 학교명 + 전화번호** 기준으로 대조
- 전화번호는 비교 전에 정규화한다.
  - 숫자 외 문자는 제거
  - 앞뒤 공백 제거
  - 국내 휴대전화 기준 11자리(`010` 시작)를 기본 정상값으로 판단
  - 정규화 실패 또는 빈 값은 자동 발급하지 않고 수동 확인 대상으로 표시
- `명단` 탭의 영수증 발급 횟수 확인
  - 발급 횟수 **0회 또는 1회** → 영수증 발급 대상
  - 발급 횟수 **2회 이상** → 제외 (이수증만 별도 처리)
- `명단` 탭의 발급 기록 정합성 검사
  - 발급횟수와 `과정명/발급날짜/링크` 기록이 서로 맞지 않으면 자동 처리하지 않고 수동 확인 대상으로 표시
  - 필수 헤더가 없거나 이름이 다르면 처리를 중단하고 누락된 헤더를 안내
- 필터링된 발급 대상자 명단을 앱 화면에 표시하여 담당자가 최종 확인
- 동명이인 또는 이름+학교명+전화번호 기준으로 유일하게 식별되지 않는 항목은 자동 발급하지 않고, 앱에서 수동 확인 대상으로 표시
- 같은 연수과정명과 같은 대상자 조합으로 이미 발급 완료된 기록이 있으면 재발급 금지 대상으로 표시
- 앱 내부 역할
  - React/TypeScript: 엑셀 로드, 미리보기, 대조 결과 표시
  - Rust/Tauri 백엔드: Google Sheets API 조회 및 대조 결과 반환

---

### 3단계 — 영수증 자동 발급 (앱 자동화)
- 2단계에서 생성된 발급 대상자 명단을 기반으로 **자체 양식**으로 영수증 자동 생성
- **파일명 규칙**: 기본은 `이름_학교명.pdf` 형식으로 저장
- 동명이인처럼 같은 이름·학교명 조합이 중복되는 경우에만 `이름_학교명_전화번호뒤4자리.pdf` 형식으로 저장
- 담당자가 발급 대상자 목록을 확인한 뒤 실행
- 초기 개발은 현재 폴더의 `receipt_sample.pdf`를 기준으로 진행
- 최종 영수증 양식 확정 후에는 템플릿 파일과 입력 좌표/필드 매핑만 교체하는 방식으로 수정

---

### 4단계 — 구글 드라이브 업로드 및 구글 시트 갱신 (앱 자동화)
- 앱 사용자가 Google 로그인으로 본인 Drive 접근 권한을 승인
- 앱에서 사용자가 지정한 Google Drive 상위 폴더 아래 연수별 폴더를 만들고, 해당 폴더에 발급된 영수증 PDF를 업로드
- 연수별 폴더명 규칙: `연수과정명_발급날짜`
- 업로드된 개별 PDF 파일은 **링크가 있는 사용자: 뷰어** 권한으로 공유 링크를 생성
- 업로드 성공 즉시 구글 스프레드시트 자동 갱신
  - `명단` 탭 갱신
  - **매칭 기준**: 이름 셀 + 학교명 셀 + 전화번호 셀이 모두 일치하는 행
  - **갱신 내용**: 영수증 발급 횟수 +1, 과정명, 발급 날짜, 개별 PDF 공유 링크 기록
- **예외 처리 및 멱등성(Idempotency) 보장**:
  - 구글 드라이브 업로드와 구글 시트 갱신은 이종 API 분산 환경이므로 완벽한 원자적 트랜잭션 처리가 불가능함.
  - 이를 보완하기 위해 앱이 로컬 작업 상태 로그(대상자 ID, 드라이브 업로드 여부, 시트 기록 여부 등)를 실시간 기록.
  - 앱 재실행 시 로컬 로그와 `명단` 탭의 발급 기록을 함께 확인하여 이미 완료된 대상자는 중복 처리하지 않음.
- **구현**: Rust/Tauri 백엔드에서 Google Drive API, Google Sheets API, 로컬 상태 로그 처리

---

## 자동화 구현 범위 요약

| 단계 | 작업 | 자동화 여부 |
|------|------|------------|
| 1 | 이수자 명단 확정 및 엑셀 공유 | ❌ 수동 |
| 2 | `명단` 탭 대조 + 발급 대상 필터링 | ✅ 앱 자동화 |
| 3 | 영수증 자동 발급 (자체 양식) | ✅ 앱 자동화 |
| 4 | 구글 드라이브 업로드 + `명단` 탭 갱신 | ✅ 앱 자동화 |
