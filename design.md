# 디자인 명세 (Design Spec)

## 디자인 철학

> **Simple · Modern · Readable**  
> 복잡한 행정 업무를 처리하는 도구인 만큼, 불필요한 시각적 요소를 제거하고  
> 사용자가 현재 단계와 결과를 즉시 파악할 수 있는 인터페이스를 지향한다.

---

## 핵심 원칙

| 원칙 | 내용 |
|------|------|
| **단순함** | 화면당 하나의 핵심 작업. 불필요한 버튼·색상·장식 요소 배제 |
| **모던함** | 둥근 모서리, 충분한 여백, 섬세한 그림자 — 과하지 않은 현대적 감각 |
| **가독성** | 충분한 폰트 크기, 명확한 계층 구조, 고대비 텍스트 |
| **상태 명확성** | 처리 중·성공·실패 상태를 항상 시각적으로 표시 |
| **오류 방지** | 비가역적 작업(발급·업로드)은 확인 단계 추가 |

| 역할 | 색상 | HEX |

## 타이포그래피

- **기본 폰트**: `Pretendard` (한국어 가독성 최적화) + 시스템 폰트 폴백
- **폰트 계층**:

| 용도 | 크기 | 굵기 |
|------|------|------|
| 페이지 제목 | 22px | 700 (Bold) |
| 섹션 제목 | 16px | 600 (SemiBold) |
| 본문 | 14px | 400 (Regular) |
| 보조 텍스트 | 12px | 400 |
| 버튼 레이블 | 14px | 500 (Medium) |

## 레이아웃

### 전체 구조
```
┌─────────────────────────────────────────┐
│  [로고]  앱 이름              [설정 아이콘] │  ← 상단 헤더 (고정)
├─────────────────────────────────────────┤
│                                         │
│          메인 컨텐츠 영역                │
│         (단계별 워크플로우)              │
│                                         │
└─────────────────────────────────────────┘
```

### 워크플로우 진행 표시
- 상단 또는 좌측에 **스텝 인디케이터** (Step 1 → 2 → 3 → 4)
- 현재 단계 하이라이트, 완료 단계 체크 표시, 미진행 단계 비활성화

### 메인 컨텐츠
- 최대 너비 `800px`, 좌우 여백으로 중앙 정렬
- 카드(Surface) 기반 섹션 구분, 카드 간 `16px` 간격
- 카드 `border-radius: 12px`, `box-shadow: 0 1px 4px rgba(0,0,0,0.08)`

---

## 로고 활용

- **위치**: 상단 헤더 좌측
- **크기**: 높이 `32px`, 너비 비율 유지
- **파일**: `logo.png`
- Primary 색상은 `logo.png`의 주색을 추출하여 버튼·강조 요소에 적용

---

## 주요 컴포넌트

### 파일 업로드 영역
- Drag-and-drop 지원 점선 테두리 박스
- 파일 선택 시 파일명·행 수 미리보기 표시
- 상태: 대기 / 드래그 오버 / 파일 선택 완료

### 데이터 테이블
- 발급 대상자 목록 테이블
- 열: 이름 / 학교명 / 전화번호 / 현재 영수증발급횟수 / 발급 여부
- 제외 대상(2회 이상)은 행을 흐리게(opacity 0.4) 표시
- 헤더 고정, 스크롤 가능

### 버튼
| 종류 | 용도 |
|------|------|
| Primary (채움) | 핵심 액션 (발급 시작, 업로드) |
| Secondary (테두리) | 보조 액션 (미리보기, 취소) |
| Destructive | 초기화 등 되돌릴 수 없는 액션 |

- 비가역적 액션(발급·업로드)은 클릭 시 확인 모달 표시

### 진행 상태 표시
- 처리 중: 진행률(%) 프로그레스 바 + 단계 설명 텍스트 ("구글 시트 대조 중...") + API Quota 대기 상태(Throttling) 시각화
- 성공: 초록 체크 아이콘 + 결과 요약 (총 대상자 수, 성공 건수, 스킵 건수 표시)
- 실패: 빨간 경고 아이콘 + 오류 상세 메시지 + **실패한 대상자 목록 표시 테이블**
- 업로드 완료 후 결과 화면에 개별 PDF가 `링크가 있는 사용자: 뷰어` 권한으로 공유되었음을 표시
- 실시간 일시정지, 이어서 진행, 실패 항목 재시도 UI는 초기 버전에서 제공하지 않는다.

### 알림 (Toast)
- 화면 우측 하단 고정
- 성공·경고·오류 3가지 타입
- 3초 후 자동 사라짐

## 화면 목록

| 화면 | 설명 |
|------|------|
| **Step 1** — 명단 로드 | 엑셀 파일 drag-and-drop 또는 선택, `이수결과`가 `"이수"`인 행만 필터링하여 미리보기 |
| **Step 2** — 대조 및 필터 | `명단` 탭 필수 헤더 검증 및 대조 실행, 발급 대상자 확인 테이블 (**이름·학교명·전화번호 불일치 또는 발급 기록 정합성 충돌 시 사용자 직접 확인용 수동 매칭 모달 제공**) |
| **Step 3** — 영수증 발급 | 발급 실행, 실시간 진행률(%), 성공·실패 결과 표시 |
| **Step 4** — 업로드 및 기록 | 사용자가 지정한 구글 드라이브 상위 폴더 아래 `연수과정명_발급날짜` 폴더 생성 + 개별 PDF 공유 링크 생성 + `명단` 탭에 과정명·발급날짜·링크 기록, 최종 결과 요약 |
| **설정** | Google 로그인 연결 상태, 스프레드시트 ID, 업로드 대상 드라이브 폴더, `명단` 탭 이름 설정 |

## 접근성

- 버튼·입력 요소 키보드 포커스 스타일 명시 (`outline`, `ring`)
- 색상만으로 상태를 전달하지 않음 (아이콘·텍스트 병행)
- 최소 클릭 영역 `44×44px` 확보
